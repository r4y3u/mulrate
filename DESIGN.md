# MulRate v1.0.0 beta.6 設計メモ

## 画面構造

画面は常に上下2分割です。

```text
┌────────────────────┐
│ ナビウィンドウ       │
│ レート・問題・結果   │
├────────────────────┤
│ テンキーウィンドウ   │
│ 手書きパッド・入力   │
└────────────────────┘
```

目的は、問題文やリザルト表示の増減でテンキー位置が動くことを防ぐことです。

## 初期画面

初期画面はレートをメイン表示にします。

- レート
- ポイント
- 出題傾向
- 学習済み
- スタート
- 学習済み
- 設定

「ポイント」は次に出る問題の具体例です。

## 入力

テンキーは3種類です。

### 普通のテンキー

```text
7 8 9
4 5 6
1 2 3
C 0 OK
```

### 1左上・0右下

```text
1 2 3 消
 Q W E C
  A S D F
   Space
```

画面上では数字表示、物理キーボードでは `Q/W/E/A/S/D/F/Space` に対応します。

### 疑似テンキー

```text
7 8 9 0
 U I O C
  J K L ;
```

物理キーボードで高速入力しやすい配置です。

## 手書きパッド

手書きパッドは文字認識ではなく、途中計算用のメモです。

- プレイ中に折りたたみ・展開可能
- レイアウト表示
- オーバーレイ表示
- テンキーとの上下入れ替え

## 正誤演出

- 正解：光のスプラッシュ
- 誤答：ナビウィンドウの軽い横揺れ
- 入力：答え欄の軽い強調
- 物理キー入力：対応テンキーの押下演出

## 出題パターン

```text
A：中心10、直前0、基礎0、挑戦0
B：中心9、直前0、基礎0、挑戦1
C：中心8、直前0、基礎0、挑戦2
D：中心5、直前3、基礎2、挑戦0
E：中心7、直前2、基礎1、挑戦0
F：中心9、直前1、基礎0、挑戦0
```

基本遷移は `A → B → C → 次類型D → E → F → A` です。

## 自動進行

全問初回正解かつ平均速度が類型基準以内の場合、確認パターンを一部スキップします。

例：

- A → C
- D → F
- E → A

自動進行時はレートに追加補正を入れます。

## 学習済み

基本確認を全問初回正解かつ十分な速度で終えた類型は「学習済み」として解放されます。

学習済み類型は、初期画面の「学習済み」から基本問題セットで反復できます。反復時の正のレート加算は通常の30%に抑えています。


## beta.3 更新

- 10問回答後、答え直し対象がない場合は自動で最終リザルトへ進むよう修正。
- テンキー名を「デフォルト」「疑似テンキー左」「疑似テンキー右」に整理。
- デフォルトの「C」を「←」表示に変更し、全テンキーで1文字削除に統一。
- 疑似テンキー左・右をキーボード配列に合わせて小型化・段差調整。
- Backspaceは1文字削除、Enter / Spaceは決定として常時利用可能。

## beta.4 更新

- 10問完了時の遷移を、誤答なしなら直接リザルト、誤答ありなら答え直し案内、に整理。
- 解答欄は問題中・答え直し中のみ表示する。
- ホーム遷移時に終了済みセットのレート処理が失われないよう保護する。


## beta.5 補正

- 高速進行は内部処理として扱い、画面上では「十分に定着」「次の確認へ進む」と表示します。
- 学習済みは、基本確認の高速全問正解だけでなく、既存データで次類型へ通過済みの類型も復元対象にします。


## beta.6 設計補正

### 学習段階の追加

2けた×2けたの導入前に、前提となる「何十でかける計算」を明示的に追加しました。

```text
M2D2_X10   ：2けた×10      例 23×10
M2D2_TENS  ：2けた×何十    例 23×20
M2D2_TEN   ：2けた×10台    例 23×12
```

この順序により、10のまとまりを使う考え方を確認してから、10台をかける筆算・暗算へ進みます。

### レート評価

速度評価だけでなく、初回正解の安定性を強めました。
高速入力でセット数を多く消化できても、誤答が残る場合はレートと学習済み解放が少し遅れます。

一方で、慎重に解いて高正答を保つ場合は、速度がやや遅くても正確性が反映されるようにしています。
