# MulRate v1.0.0 設計メモ

## 基本思想

MulRate v1.0.0 は、オンライン要素を入れず、1人用の学習ループを固めるための MVP です。

中心は次の3点です。

1. レートを進度表示の中心にする
2. 学習内容を細分化し、系統的に出題する
3. 速く正しく解くことと、ミスを自力で直すことの両方を評価する

## 初期画面の表示

表示は簡略化しています。

```text
レート：3240
ポイント：3×4
```

`ポイント` は、次に中心となる問題類型を具体例で示すための表示です。抽象的な「現在レベル」は表示しません。

## 画面フロー

```text
初期画面
  ├─ 問題画面
  │   ├─ ポーズ画面
  │   │   ├─ 問題画面へ
  │   │   └─ 初期画面へ
  │   └─ リザルト画面
  │       ├─ 答え直し
  │       ├─ 問題画面へ
  │       └─ 初期画面へ
  └─ 設定画面
      └─ 初期画面へ
```

ランキング画面は v1.0.0 から外しています。

## 問題画面とリザルト画面

問題画面・リザルト画面・答え直し画面は同じ入力エリアを共有します。

```text
上部ステータス
メイン表示エリア
手書きパッド
テンキー
```

手書きパッドは Canvas メモです。文字認識はしません。

## 出題比率パターン

10問を1セットとし、比率パターンを A〜F で管理します。

| パターン | 中心 | 直前 | 基礎 | 挑戦 |
|---|---:|---:|---:|---:|
| A | 10 | 0 | 0 | 0 |
| B | 9 | 0 | 0 | 1 |
| C | 8 | 0 | 0 | 2 |
| D | 5 | 3 | 2 | 0 |
| E | 7 | 2 | 1 | 0 |
| F | 9 | 1 | 0 | 0 |

進み方：

```text
A → B → C → 次の類型へ
D → E → F → A → B → C → 次の類型へ
```

全問初回正解かつ平均時間が目標値の 1.15 倍以内なら、次のパターンへ進みます。
結果が悪い場合は前のパターンへ戻ります。
それ以外は同じパターンにとどまり、正のレート変動は維持回数に応じて半減していきます。

## 問題生成方式

完全な固定問題データベースではなく、次のハイブリッド方式です。

```text
類型データベース + アルゴリズム生成
```

類型データベースが持つ情報：

- 類型ID
- 表示名
- ポイント例
- 難度係数
- 目標回答時間
- 生成関数

実際の問題は、出題時にアルゴリズムで生成します。
直近の出題と重なりにくくするため、最近出した式は localStorage に保存します。

## v1.0.0 の実装済み問題類型

### 九九

- 九九 5・2の段
- 九九 3・4の段
- 九九 6の段
- 九九 7・8の段
- 九九 9・1の段
- 九九 混合

### 2けた×1けた

- 繰り上がりなし
- 一の位で繰り上がり
- 十の位で繰り上がり
- 複数回の繰り上がり
- 被乗数の一の位が0

### 3けた×1けた

- 繰り上がりなし
- 繰り上がりあり
- 連続した繰り上がり
- 被乗数に0を含む

### 2けた×2けた

- 10台をかける
- 繰り上がり少なめ
- 繰り上がりあり
- 0を含む計算
- 混合

## 今後の細分化候補

v1.0.0 では整数の乗法を優先しています。今後は次の方向に拡張できます。

- 10倍・100倍・1000倍
- 倍数・約数につながる乗法
- 小数×整数
- 小数×小数
- 分数×整数
- 分数×分数
- 交換法則・結合法則・分配法則を使う計算
- 面積・割合・比例と結びつく乗法場面

## レート表示

カンマ区切りは使いません。4桁区切りで薄い空白を入れます。

```text
99999999 → 9999 9999
1234567  → 123 4567
3240     → 3240
```

## レート計算

1問ごとの評価は次の要素を使います。

- 初回正解
- 答え直し正解
- 最終不正解
- 回答速度
- 類型難度

答え直し正解は、初回正解と同じ評価にはしません。v1.0.0 では係数 0.45 としています。

レート変動には次の補正をかけます。

- 高難度類型ほど変動を大きくする
- 現在レートが高いほど伸びを抑える
- 同じパターン維持時は正の変動を半減させる
- 最終不正解が多い場合は減点する

## レートの信頼性

v1.0.0 は完全クライアントサイドなので、レートは「端末内の学習記録」です。
将来、他者比較に耐える指標にするには、サーバー側で次の対策が必要です。

- 問題生成のサーバー管理
- 回答ログの署名
- 結果改ざん防止
- 連続回答・異常速度検知
- アカウントまたは匿名ID管理
- 学校利用時の個人情報最小化
