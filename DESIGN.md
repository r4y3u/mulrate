# MulRate v1.0.0 beta.2 設計メモ

## 画面構造

画面は常に上下2分割です。

```text
┌────────────────────┐
│ ナビウィンドウ       │
│ レート・問題・結果   │
├────────────────────┤
│ テンキーウィンドウ   │
│ 手書きパッド・入力   │
└────────────────────┘
```

目的は、問題文やリザルト表示の増減でテンキー位置が動くことを防ぐことです。

## 初期画面

初期画面はレートをメイン表示にします。

- レート
- ポイント
- 出題傾向
- 学習済み
- スタート
- 学習済み
- 設定

「ポイント」は次に出る問題の具体例です。

## 入力

テンキーは3種類です。

### 普通のテンキー

```text
7 8 9
4 5 6
1 2 3
C 0 OK
```

### 1左上・0右下

```text
1 2 3 消
 Q W E C
  A S D F
   Space
```

画面上では数字表示、物理キーボードでは `Q/W/E/A/S/D/F/Space` に対応します。

### 疑似テンキー

```text
7 8 9 0
 U I O C
  J K L ;
```

物理キーボードで高速入力しやすい配置です。

## 手書きパッド

手書きパッドは文字認識ではなく、途中計算用のメモです。

- プレイ中に折りたたみ・展開可能
- レイアウト表示
- オーバーレイ表示
- テンキーとの上下入れ替え

## 正誤演出

- 正解：光のスプラッシュ
- 誤答：ナビウィンドウの軽い横揺れ
- 入力：答え欄の軽い強調
- 物理キー入力：対応テンキーの押下演出

## 出題パターン

```text
A：中心10、直前0、基礎0、挑戦0
B：中心9、直前0、基礎0、挑戦1
C：中心8、直前0、基礎0、挑戦2
D：中心5、直前3、基礎2、挑戦0
E：中心7、直前2、基礎1、挑戦0
F：中心9、直前1、基礎0、挑戦0
```

基本遷移は `A → B → C → 次類型D → E → F → A` です。

## 飛び級

全問初回正解かつ平均速度が類型基準以内の場合、確認パターンを一部スキップします。

例：

- A → C
- D → F
- E → A

飛び級時はレートに追加補正を入れます。

## 学習済み

パターンAを全問初回正解かつ十分な速度でクリアした類型は「学習済み」として解放されます。

学習済み類型は、初期画面の「学習済み」からパターンAで反復できます。反復時の正のレート加算は通常の30%に抑えています。
