# MulRate v2.0.0 alpha.2 テスト報告

## 静的検査

- `app.js`、`name-filter.js`、`online-config.js`、`online-adapter.js` のJavaScript構文検査。
- Supabase Edge Functionと共有名前フィルターのTypeScript構文変換検査。
- HTMLのID重複検査。
- JavaScriptから参照する静的DOM IDの存在検査。
- Supabase SQLのテーブル、索引、RLS、トランザクション関数の構成確認。

## Chromium自動操作検査

- 空の保存領域でプロフィール画面が必須表示になること。
- 必須設定中はホームボタンと戻るボタンから回避できないこと。
- 禁止語を含むニックネームが拒否されること。
- 正常なニックネームを保存するとホームへ進むこと。
- 保存後のニックネーム入力欄が無効化されること。
- 初回に同意しなかった場合、設定画面の同意チェックが有効であること。
- 設定画面で同意すると確認後にチェックが固定されること。
- 全データリセット後、名前・同意・学習データが消え、プロフィール必須画面へ戻ること。
- オンライン未接続時もランキングオーバーレイを開閉できること。
- 模擬オンラインアダプターで `37位／512位中` と表示されること。
- 模擬データ200件をランキングオーバーレイへ表示できること。
- 現在プレイヤー行が識別表示されること。
- ナビウィンドウ上の手書きパッドで正解時に大きな `○` が表示されること。
- キーパッド上オーバーレイの手書きパッドで誤答時に大きな `×` が表示されること。
- 上記操作中にJavaScript実行エラーが発生しないこと。

## 名前フィルターの誤判定確認

次の一般語は許可されることを確認した。

- ポケモン
- シネマ
- コロスケ
- アナリスト
- grass

完全一致の短い禁止語、区切り記号を挟んだ代表的回避表記、管理者詐称名は拒否されることを確認した。

## 実環境で追加確認したい項目

- iPhone Safari / Braveで初回プロフィール画面が見切れないこと。
- Android Chromeで手書きパッド上の `○` `×` が入力を妨げないこと。
- 実際のSupabaseプロジェクトで200件取得と順位算出が一致すること。
- 回線切断中の送信失敗後も通常学習を続けられること。
- オンラインデータ削除失敗時にローカルデータが保持されること。
- v1およびalpha.1を使用済みの実端末で移行結果が適切であること。
