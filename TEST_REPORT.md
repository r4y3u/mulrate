# MulRate v2.0.0 alpha.3 テスト報告

## 静的検査

- `app.js`、`name-filter.js`、`online-config.js`、`online-adapter.js` のJavaScript構文検査。
- HTMLのID重複検査。
- JavaScriptから参照する主要DOM IDの存在検査。
- Supabase Edge Functionの端末秘密トークン検査、Origin制限、認証付き削除処理を確認。
- SQLマイグレーションの認証ハッシュ列、認証付きコミット関数、削除関数、送信回数制限を確認。

## Chromium自動操作検査

- 空の保存領域ではプロフィール画面だけが表示され、設定の歯車が出ないこと。
- プロフィール決定後、初期画面の操作ボタンが「スタート」「学習済み」「記録・ランキング」の3つだけであること。
- 初期画面にプロフィール・設定の大きなボタンが存在しないこと。
- 通常画面では右上に設定の歯車が表示され、ポーズボタンは非表示であること。
- 回答中はポーズボタンだけが表示され、設定の歯車が非表示であること。
- ポーズ画面では設定の歯車が表示されること。
- ホーム、ランキング、ポーズから設定を開き、`× 閉じる` で元の画面へ戻ること。
- 設定表示中は数字キーやEnterで背後の問題へ入力されないこと。
- 端末秘密トークンが32文字以上で生成され、ランキング送信データへ含まれること。
- 模擬通信失敗時に完了セットが送信待ちキューへ残ること。
- onlineイベント後の模擬成功でキューが消え、順位と参加人数が更新されること。
- 上記操作中にJavaScript実行エラーが発生しないこと。

## 継続確認項目

- iPhone Safari、Android Chromeで設定オーバーレイの本文末尾までスクロールできること。
- 実Supabase環境でalpha.2既存プレイヤーへ端末トークンを初回登録できること。
- 異なる秘密トークンによる同一端末IDの更新・削除が拒否されること。
- 1分30件を超える送信が拒否され、通常学習は継続できること。
- 複数の未送信セットが順番どおり再送されること。
- 本番公開前にレート式全体をサーバーで再計算できること。
