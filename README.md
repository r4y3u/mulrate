# MulRate v2.0.0 alpha.2

乗法に特化したレート制トレーニングアプリです。v1.0.0 beta.19を基礎に、プロフィール固定、オンラインランキングUI、手書きパッド上の正誤表示、Supabase接続の土台を追加しています。

## alpha.2の主な更新

### プロフィール

- 初回起動時とアプリデータのリセット直後は、プロフィール設定を完了するまで他画面へ進めない。
- ニックネームは1〜12文字。保存後はアプリデータをリセットするまで変更不可。
- 第8世代以降のポケモン系入力制限を参考にした禁止語判定を追加。
  - Unicode正規化、ひらがな・カタカナの同一視、区切り記号を使った回避、代表的な英字置換を判定する。
  - 短い英単語は完全一致に限定し、無関係な名前を巻き込む部分一致を抑える。
  - 運営・管理者を装う表示名も拒否する。
- ランキング公開への同意は任意。
- 同意しなかった場合は、設定画面から後で同意可能。
- 一度同意すると、アプリデータをリセットするまで解除不可。

参考: https://wiki.pokemonwiki.com/wiki/%E7%A6%81%E6%AD%A2%E3%83%AF%E3%83%BC%E3%83%89%E4%B8%80%E8%A6%A7

### 記録・ランキング

- オンライン接続時は「現在順位／参加人数」を `○位／○○位中` 形式で表示。
- オンラインランキング枠をホバー・フォーカス・クリックに対応。
- クリックするとトップ200のランキングをオーバーレイ表示。
- 一覧はスクロール可能。オーバーレイ内のクリックまたはEscapeキーで閉じる。
- 未接続時も同じ導線を表示し、接続前のUI確認ができる。

### 手書きパッド

- 手書きパッドを開いたまま回答した場合、パッド上へ大きな `○` または `×` を表示。
- ナビウィンドウ上配置とキーパッド上オーバーレイ配置の両方に対応。

### オンライン接続の次段階

- `online-config.js` と `online-adapter.js` をSupabase Edge Function対応へ更新。
- Supabase用SQLマイグレーションとEdge Function雛形を追加。
- セッション重複、問題と回答の積、正誤、時間、ダイジェスト、レート前後関係をサーバー側で基本検証する。
- 既存プレイヤーの表示名変更をサーバー側でも拒否する。
- アプリデータのリセット時、接続済みならオンライン側の削除成功後にローカルデータを消去する。

この版のサーバー検証は `basic` 段階です。公開ランキングを本番運用する前に、レート計算そのものをサーバーへ移植する必要があります。

## 現在のオンライン動作

`online-config.js` は空欄のため、配布状態では外部サーバーへ送信しません。Supabaseを接続する手順は `SUPABASE_SETUP.md` を参照してください。

## 実行方法

`index.html` をブラウザで開きます。通常の学習機能は外部ライブラリを必要としません。

GitHub Pagesへ配置する場合は、少なくとも次のファイルを同じ階層へ置いてください。

- `index.html`
- `style.css`
- `app.js`
- `name-filter.js`
- `online-config.js`
- `online-adapter.js`

## データ移行

次の保存キーを検出すると、alpha.2の保存構造へ自動移行します。

- `mulrate_v2_0_0_alpha1_state`
- `mulrate_v1_0_0_beta18_state`

alpha.1で独自のニックネームを保存済みの場合は固定プロフィールとして引き継ぎます。未設定の既定名しかない場合は、初回プロフィール設定を表示します。
