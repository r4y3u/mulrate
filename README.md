# MulRate v2.0.0 alpha.4

乗法に特化したレート制トレーニングアプリです。v1.0.0 beta.19を基礎に、固定プロフィール、端末内記録、オンラインランキング接続基盤、手書きパッド上の正誤表示を段階的に追加しています。

## alpha.4の主な更新

### 表記変更

オンラインランキングの同意欄を、次の表記へ変更しました。

- `ニックネームと成績をランキングに送信・公開することに同意する。`
- 同意後の説明: `同意済みです。解除するためには、アプリデータのリセットが必要です。`

### サーバー側レート検証

alpha.4のEdge Functionは、クライアントが送ったレート増減をそのまま採用しません。次の情報から、サーバー側でレートを再計算します。

- 進行中の類型と出題パターン
- パターン停滞回数
- 前回オンライン送信からの進行段階の連続性
- 通常学習／学習済み反復の別
- 10問の類型構成
- 初回正誤、答え直し後の正誤
- 各問題の初回答時間
- セット開始時レート

サーバー計算と、クライアントの結果・進行判定・レート増減が一致しない送信は拒否します。2回目以降は、前回サーバー保存時の類型・パターン・停滞回数から連続していない送信も拒否します。保存されるレート増減も、Edge Functionが再計算した値です。

### 問題構成の検証

- 通常問題は、進行段階とA〜Fパターンから期待される類型数を照合します。
- 九九は、段・乗数範囲・10問構成を照合します。
- 学習済み反復は、10問が同じ指定類型であることを照合します。
- 類型番号、類型ID、パターン番号、パターンIDの不一致を拒否します。

### データ移行

次の保存キーを検出するとalpha.4形式へ移行します。

- `mulrate_v2_0_0_alpha3_state`
- `mulrate_v2_0_0_alpha2_state`
- `mulrate_v2_0_0_alpha1_state`
- `mulrate_v1_0_0_beta18_state`

alpha.3以前の未送信キューにはサーバー再計算に必要な進行情報がないため、移行時に対象外とします。端末内の学習履歴・レート・設定は維持されます。

## 現在のオンライン動作

`online-config.js`は空欄です。配布状態では外部通信は発生しません。接続方法は`SUPABASE_SETUP.md`を参照してください。

alpha.4では各セットの増減をサーバーで検証します。ただし、既存端末が初めてオンラインへ参加するときの開始レートと進行段階は、端末内データを基準にします。初回基準値を完全に証明する仕組みは未実装ですが、初回送信後のレートと進行はサーバーで連続性を管理します。本番ランキングでは、新規オンライン参加を300開始に限定するか、既存データを暫定扱いにする運用が必要です。

## 実行方法

`index.html`をブラウザで開きます。通常の学習機能は外部ライブラリを必要としません。

GitHub Pagesへ配置する場合は、次のファイルを同じ階層へ置きます。

- `index.html`
- `style.css`
- `app.js`
- `name-filter.js`
- `online-config.js`
- `online-adapter.js`
